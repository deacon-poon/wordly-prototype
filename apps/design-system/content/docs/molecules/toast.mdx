# Toast

A brief, non-blocking notification that appears temporarily and auto-dismisses. Use for confirming actions, reporting success, or surfacing non-critical information.

## Variants

| Variant | Description |
|---------|-------------|
| Light | Default light mode appearance |
| Dark | Dark mode appearance |
| Default | Standard informational toast |
| Success | Positive confirmation |
| Error | Action failure notification |
| With action | Includes an undo or follow-up action button |

## Usage Guidelines

- Use for transient feedback that does not require user interaction
- Keep messages short and direct (e.g., "Changes saved" not "Your changes have been successfully saved")
- Position toasts consistently (typically bottom-right or top-right)
- Ensure toasts are announced to screen readers via `role="status"` or a live region
