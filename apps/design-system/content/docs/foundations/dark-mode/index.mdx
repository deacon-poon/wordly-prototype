# Dark Mode

This document defines how the Wordly design system adapts to dark mode, starting with mobile. The approach is informed by Material Design 3, Apple HIG, and Airbnb's design systems.

## Core Principles

1. **Dark, not black.** The base surface is `#121416` (gray-900), not `#000000`. Pure black creates excessive contrast and halation on OLED screens.
2. **Elevation = lighter.** Higher surfaces are progressively lighter, replacing shadows that are invisible on dark backgrounds.
3. **Brand stays recognizable.** The Wordly blue shifts lighter (500 → 300) for sufficient contrast against dark surfaces, but hue remains constant.
4. **Mixed-mode zones are allowed.** Branded regions (the blue gradient header, logo lockup) stay in light mode. Content cards go dark. A clear boundary separates them.

### Construction

The dark surfaces are not pure grays — they carry a faint blue tint from brand-blue to maintain brand warmth:

| Level | Base gray | + brand-blue tint | Result |
|-------|-----------|-------------------|--------|
| Base | `#121416` | 0% | `#121416` |
| Elevated 1 | `#1a1c1e` | 3% brand-blue-900 | `#1a1d21` |
| Elevated 2 | `#212325` | 5% brand-blue-900 | `#21252b` |
| Elevated 3 | `#282a2c` | 7% brand-blue-800 | `#282d35` |

## Brand Color Adaptation

The Wordly blue shifts **2–3 steps lighter** in dark mode to maintain vibrancy and WCAG AA contrast against dark surfaces. Hue (211) stays constant.

<div className="not-prose flex flex-col gap-3 mb-8">
  <ModePair name="Brand" light="#017cff" dark="#3396ff" token="brand/brand" />
  <ModePair name="Primary Action" light="#0063cc" dark="#3396ff" token="actions/primary" />
  <ModePair name="Hover" light="#0051a8" dark="#75b8ff" token="actions/hover" />
  <ModePair name="Active" light="#00458f" dark="#b2d8ff" token="actions/active" />
  <ModePair name="Brand Background" light="#f0f7ff" dark="#001e3d" token="brand/bg" />
</div>

### Why lighter, not the same?

| | Light mode primary on white | Dark mode primary on #121416 |
|---|---|---|
| `#0063cc` (500) | 4.7:1 contrast ✓ | 3.2:1 ✗ fails AA |
| `#3396ff` (300) | 2.4:1 ✗ | 5.8:1 ✓ passes AA |

The 300 step passes AA on dark surfaces while the 500 step does not. This follows the Material Design 3 pattern of using tone 80 (lighter) in dark mode vs tone 40 (darker) in light mode.

## Mixed-Mode Pattern: Mobile Attendee Screen

The Wordly mobile attendee screen demonstrates a **split-mode** layout — a branded light zone and a dark content zone.

### What stays light

| Element | Reason | Token |
|---------|--------|-------|
| Top gradient background | Brand-blue-50 `#d6eaff` — this IS the brand identity | Stays `brand-blue-50` in both modes |
| Logo container | White circle with Wordly icon — brand lockup | Stays `#ffffff` |
| "AI Powered Translation" text | Sits on the light brand background | Stays `#0b1c3a` (navy) |

### What goes dark

| Element | Light | Dark | Token |
|---------|-------|------|-------|
| Card background | `#ffffff` | `#1a1d21` (surface/1) | `card/bg` |
| Card border radius | 16px | 16px | unchanged |
| Card shadow | `0 2px 48px rgba(0,0,0,0.02)` | `none` (surfaces differentiate by color) | `card/shadow` |
| "Attend a session" heading | `#121416` | `#f0f2f4` | `text/primary` |
| "Session ID" label | `#121416` | `#f0f2f4` | `text/primary` |
| Input placeholder | `#646e78` | `#495057` | `text/placeholder` |
| Input border | `#e2e8f0` | `rgba(255,255,255,0.16)` | `input/border` |
| Input background | `#ffffff` | `#21252b` (surface/2) | `input/bg` |
| "Join" button background | `#0063cc` | `#3396ff` | `button/primary-bg` |
| "Join" button text | `#f8fafc` | `#001e3d` | `button/primary-text` |
| "Back" button background | `#ffffff` | `#21252b` | `button/secondary-bg` |
| "Back" button text | `#020617` | `#f0f2f4` | `button/secondary-text` |
| "Back" button border | `#e2e8f0` | `rgba(255,255,255,0.16)` | `button/secondary-border` |

### The boundary

The transition from light header to dark card is natural — the card already has `border-radius: 16px` and overlaps the gradient background. In dark mode:
- The card surface (`#1a1d21`) creates a clear boundary against the light blue gradient
- No additional separator element is needed
- The card's rounded corners provide the visual "zone switch"

## Implementation Checklist

- [ ] Define CSS custom properties with light/dark values using `prefers-color-scheme` or a class toggle
- [ ] Surface elevation tokens: base through elevated-4
- [ ] Semi-transparent borders — avoid hardcoded opaque grays
- [ ] Brand-blue shifts to 300 step for primary actions
- [ ] Text uses off-white (`#f0f2f4`), never pure white for body text
- [ ] Status color backgrounds invert to 900-step values
- [ ] Shadows removed on dark surfaces — rely on color differentiation
- [ ] Input fields use surface/2 background to show nesting
- [ ] Test all combinations at WCAG AA (4.5:1 body text, 3:1 large text)
